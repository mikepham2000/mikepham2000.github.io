File Format Quick-Reference Data files from PLINK (a.k.a bfile)

*Important note: These files should already passed the QC from the genotype company (GenCall Rate from Illumina) if not it can not be used.

data.fam (Sample Information)

Each row = one individual. Six columns, space/tab-delimited:

ColumnFieldDescription
1FIDFamily ID
2IIDIndividual ID (within family)
3Father IDPaternal ID (0 if unknown)
4Mother IDMaternal ID (0 if unknown)
5Sex1 = male, 2 = female, 0 = unknown
6Phenotype-9 = missing (default)

data.bim (Variant Information)

Each row = one variant. Six columns:

ColumnFieldDescription
1CHRChromosome code
2SNPVariant ID (e.g., rs number)
3cMGenetic distance in Morgans (often 0)
4BPBase-pair coordinate
5A1Allele 1 (usually minor)
6A2Allele 2 (usually major)

data.bed (Binary Genotype File)

  • Binary-encoded genotype data — not human-readable
  • Always accompanied by .bim and .fam files
  • These three files together are called bfiles (binary fileset)

data.ped (Text Genotype File)

First 6 columns = same as .fam. Remaining columns = genotype data (two allele columns per SNP).

  • Paired with a .map file
  • Not recommended for new scripts — always convert to binary (.bed) for efficiency

data.map

Four columns: CHR, SNP ID, cM, BP. This is the text-format companion to .ped.

data.fam - information about all individual

-9 is encoded for missing or unknown by default

Strand Flipping & Allele Issues

Strand inconsistencies are one of the most common problems when merging data from different genotyping platforms. PLINK has several tools to handle this.

Flip strand for A/C/G/T SNPs

./plink19 --bfile mydata --flip snplist.txt --make-bed --out mydata_flipped

This swaps A↔T and C↔G for all SNPs in the list.

Trial Flip Workflow (Resolving Merge Conflicts)

When a merge fails due to allele mismatches, PLINK writes a .missnp file. Use this workflow:

# Step 1: Try merging — it fails, produces merged.missnp
./plink19 --bfile source1 --bmerge source2 --make-bed --out merged

# Step 2: Flip problematic SNPs in source2
./plink19 --bfile source2 --flip merged.missnp --make-bed --out source2_trial

# Step 3: Retry merge
./plink19 --bfile source1 --bmerge source2_trial --make-bed --out merged_trial

# Step 4: If most errors disappeared → strand issue confirmed
# Flip remaining errors back
./plink19 --bfile source2_trial --flip merged_trial.missnp --make-bed --out source2_corrected

# Step 5: Final merge
./plink19 --bfile source1 --bmerge source2_corrected --make-bed --out merged_final

Update Allele Codes

When you need to convert allele coding (e.g., from A/B to A/C/G/T using manifest data):

./plink19 --bfile mydata --update-alleles allele_update.txt --make-bed --out mydata_nt

The allele_update.txt file has 5 columns: SNP ID, Old A1, Old A2, New A1, New A2.

Merging Datasets

Helpful for multi-breed quality control and merge afterwards

Merge Two Binary Filesets

./plink19 --bfile source1 --bmerge source2 --make-bed --out merged

Merge Multiple Filesets

Create a text file (merge_list.txt) with one fileset prefix per line:

source2
source3
source4
./plink19 --bfile source1 --merge-list merge_list.txt --make-bed --out merged_all

Handling Triallelic Errors

If the merge fails due to 3+ alleles at a locus, PLINK writes merged.missnp. To exclude and proceed:

./plink19 --bfile source1 --exclude merged.missnp --make-bed --out source1_clean
./plink19 --bfile source2 --exclude merged.missnp --make-bed --out source2_clean
./plink19 --bfile source1_clean --bmerge source2_clean --make-bed --out merged

Merge Modes

ModeBehavior
1 (default)Ignore missing, set mismatches to missing
2Only overwrite missing calls in original
3Only overwrite with nonmissing calls from new file
4Never overwrite
5Always overwrite
6Report all mismatches (no merge)
7Report nonmissing mismatches only (no merge)
./plink19 --bfile source1 --bmerge source2 --merge-mode 5 --make-bed --out merged

LD Pruning

LD pruning removes correlated variants. Essential before PCA, GRM construction, or IBD estimation.

Mostly for human genotypes and may be for whole genome sequencing data

Pairwise LD Pruning (Most Common)

# Window size 50 SNPs, step 5 SNPs, r² threshold 0.2
./plink19 --bfile mydata --indep-pairwise 50 5 0.2 --out pruned

# Apply the pruning
./plink19 --bfile mydata --extract pruned.prune.in --make-bed --out mydata_pruned

Outputs:

  • pruned.prune.in — SNPs to keep
  • pruned.prune.out — SNPs to remove

Compute LD Between Variant Pairs

# Pairwise r² for all SNPs within 1000 kb
./plink19 --bfile mydata --r2 --ld-window-kb 1000 --ld-window 99999 --ld-window-r2 0.2 --out ld_results

PCA & Population Structure Visualization

PCA

# Compute top 10 PCs (default = 20)
./plink19 --bfile mydata_pruned --pca 10 --out pca_results

Outputs:

  • pca_results.eigenval — eigenvalues
  • pca_results.eigenvec — eigenvectors (sample PCs), columns: FID, IID, PC1, PC2, …

MDS Plot (Alternative to PCA)

./plink19 --bfile mydata_pruned --cluster --mds-plot 4 --out mds_results
💡 Always LD-prune before running PCA or MDS for dense pannel

Filtering Cheat Sheet

Per-Variant Filters

FlagWhat It DoesTypical Threshold
--maf 0.01Remove rare variants (MAF < 1%)0.01–0.05
--geno 0.05Remove SNPs missing in >5% of samples0.02–0.10
--hwe 1e-6Remove SNPs failing Hardy-Weinberg test1e-6 (cases), 1e-4 (controls)
💡 For Crossbred population, hwe will not hold. Should NOT be CHECKED

Per-Individual Filters

FlagWhat It DoesTypical Threshold
--mind 0.05Remove individuals missing >5% of SNPs0.02–0.10

Subsetting

FlagWhat It Does
--extract snplist.txtKeep only SNPs in file
--exclude snplist.txtRemove SNPs in file
--keep samples.txtKeep only samples in file (FID + IID)
--remove samples.txtRemove samples in file (FID + IID)
--chr 1-22Restrict to autosomes only

Typical QC Pipeline Order

# Step 1: Per-individual missingness
./plink19 --bfile raw --mind 0.05 --make-bed --out qc1

# Step 2: Per-SNP missingness
./plink19 --bfile qc1 --geno 0.05 --make-bed --out qc2

# Step 3: MAF filter
./plink19 --bfile qc2 --maf 0.01 --make-bed --out qc3

# Step 4: HWE filter
./plink19 --bfile qc3 --hwe 1e-6 --make-bed --out qc4

# All at one
./plink19 --bfile qc3 --geno 0.05 --maf 0.01 --mind 0.05 --hwe 1e-6 --make-bed --out qc4

################################################################################
################### OPTIONAL ###################################################################################################################################
# Step 5: LD prune (for PCA/GRM only, not for final analysis data)
./plink19 --bfile qc4 --indep-pairwise 50 5 0.2 --out prune
./plink19 --bfile qc4 --extract prune.prune.in --make-bed --out qc4_pruned


# Step 6: Check heterozygosity (flag outliers manually)
./plink19 --bfile qc4_pruned --het --out het_check

# Step 7: Check relatedness (IBD)
./plink19 --bfile qc4_pruned --genome --min 0.185 --out ibd_check

VCF ↔ PLINK Conversion

Import VCF to PLINK Binary

./plink19 --vcf input.vcf.gz --make-bed --out mydata

Export PLINK to VCF

# Using merged FID_IID as sample ID
./plink19 --bfile mydata --recode vcf --out output

# Using only IID as sample ID
./plink19 --bfile mydata --recode vcf-iid --out output

# Gzipped output
./plink19 --bfile mydata --recode vcf-iid bgz --out output
⚠️ When exporting to VCF with correct reference alleles, include:
./plink19 --bfile mydata --a2-allele ref_alleles.txt --real-ref-alleles --recode vcf-iid --out output

Scrape Allele Codes from a VCF

./plink19 --bfile mydata --a2-allele input.vcf 4 3 '#' --make-bed --out mydata_refcorrected

Common Problem

1. The AD Recoding Trap

Using --recode AD produces additive + dominance columns, so the number of columns doubles compared to --recode A. The dominance column encodes heterozygotes as 1, everything else as 0.

# Additive only (0/1/2) — most common use case
./plink19 --bfile mydata --recode A --out geno012

# Additive + Dominance — 2× the columns!
./plink19 --bfile mydata --recode AD --out geno012_AD

2. -keep-allele-order Is Critical

By default, PLINK reassigns the major allele as A2 on every --make-bed. This silently changes your allele encoding. Always use --keep-allele-order when allele assignment matters:

./plink19 --bfile mydata --keep-allele-order --make-bed --out mydata_preserved

3. -nonfounders for Datasets Without Pedigree

If all individuals have Father = 0 and Mother = 0, PLINK treats everyone as founders and skips some calculations (like --freq and --hardy). Use --nonfounders to force inclusion:

./plink19 --bfile mydata --nonfounders --freq --out freq_results

4. Missing Genotype Encoding: Input vs Output

These are two different flags:

  • -missing-genotype - → tells PLINK that means missing in the input file
  • -output-missing-genotype 5 → tells PLINK to write 5 for missing in the output file

They are independent and often both needed:

./plink19 --lfile geno --missing-genotype - --output-missing-genotype 5 --make-bed --out geno

5. -allow-no-sex for Datasets Without Sex Info

PLINK removes individuals with ambiguous sex by default in some analyses. If your data doesn't have sex information:

./plink19 --bfile mydata --allow-no-sex --assoc --out results

6. Chromosome Coding in Output

Different tools expect different chromosome codes (e.g., 23 vs chrX vs X). Control this with:

./plink19 --bfile mydata --output-chr chrM --recode vcf --out output
# Options: 26, M, MT, 0M, chr26, chrM, chrMT

7. X Chromosome Pseudo-Autosomal Region

Most heterozygous haploid warnings come from the PAR region not being split off. Fix with:

./plink19 --bfile mydata --split-x b37 --make-bed --out mydata_splitx
# Use b36, b37, or b38 depending on your genome build

PLINK 1.9 vs PLINK 2.0 — When to Use Which

FeaturePLINK 1.9PLINK 2.0
Binary format.bed/.bim/.fam.pgen/.pvar/.psam
Multiallelic variantsNot supported (drops to biallelic)Supported natively
Variant ID templates--set-missing-var-ids (A1/A2 based)--set-all-var-ids (REF/ALT based — more reliable)
Allele orderMajor = A2 by default (confusing)Reference allele preserved properly
Dosage dataLimited (--dosage)Full support
SpeedFastFaster (especially for large datasets)
KING kinshipNot available--make-king

Use PLINK 2.0 when:

  • Working with multiallelic variants
  • You need REF/ALT-aware variant naming
  • Handling dosage/imputed data
  • Processing very large WGS datasets

Stick with PLINK 1.9 when:

  • Your pipeline or downstream tool (e.g., BLUPF90) expects .bed/.bim/.fam
  • You need -recode modifiers not yet ported to 2.0
  • Legacy script compatibility
